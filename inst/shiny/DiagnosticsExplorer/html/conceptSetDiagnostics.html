Description

A table showing the concepts included in a concept set of a cohort along with concepts recommended for review. Record and database counts represent counts collected across participating ConceptPrevalence datasources and do not represent your datasource counts. Concept counts and database counts with descendants reflect the total counts of a concept with its descendants.

Recommended concepts include (Concept in Set):

  • Not included - parent: a list of all immediate ancestors of the concepts in a concept set;
  • Not included - descendant: a list of all descendants of the concepts in a concept set, which were not included in this concept set;
  • Not included - recommended via standard: a list of recommendations of standard concepts not included in a concept set;
  • Not included - recommended via source: a list of recommendations of concepts not included in a concept set, which were found through not included source concepts (e.g. ICD-10 CM);

Source concepts are identified in the _source_concept_id fields of the Common Data Model, (e.g. drug_source_concept_id) and are used to identify the specific source codes used in a database. Standard concepts are found using the _concept_id fields (e.g. drug_concept_id), and use the same coding system across all databases.

Options

Select the cohort and the specific concept set within that cohort to explore in the side bar.

You can switch between Source Concepts and Standard Concepts at the top of the table.

What to look for

  • Are there included codes that should not be? For example, in a concept set for hypertensive disorder, are hypotension codes included by accident?
  • Are there parent or descendant concepts that should be included? For example, in a concept set for hypertensive disorder, Chronic kidney disease due to hypertension is a suggested parent that may be included.
  • Are there recommended concepts that should be included in the concept set?
